Ce document a pour but de vous faciliter le travail sur votre interface avec l'API EnvoiMoinsCher.com

Progressivement on exploitera les éléments qui vous permettront de développer un module e-shipping dans votre e-boutique. Dans le premier exemple on va récupérer les catégories. Ensuite on passera à la gestion des pays pour finir avec le devis et la passation de commande.

Le contenu de la librairie

Le package contient 6 répertoires :

  • documentation - la documentation de la librairie
  • env - les classes qui permettent l'interaction avec l'API
  • lib - le répertoire où vous pouvez placer vos classes
  • samples - les fichiers avec les exemples d'utilisation de la librairie
  • test - un fichier qui teste l'apptitude de votre environnement par rapport à la librairie
  • utils - le dossier avec des fonctions utilisées par les exemples placés dans le répertoire samples/

1. Comment obtenir la liste des catégories ?

Grâce à l'API vous pouvez obtenir la liste des contenus qui vont vous servir pour le développement de votre module.

<?php
  • $contentCl = new Env_ContentCategory(array("user" => "login", "pass" => "mot_de_passe", "key" => "cle_api"));
  • $contentCl->getContents();
?>

L'API a besoin du code de la catégorie de contenu, représenté par la variable $contents.

Regardez aussi les exemples : récupération des catégories

2. Comment obtenir la liste des pays ?

Les commandes chez EnvoiMoinsCher.com sont réalisées avec les codes ISO des pays. Pour l'instant le système gère uniquement les commandes passées depuis la France vers l'étranger. La récupération des pays se présente ainsi :

<?php
  • $countryCl = new Env_Country(array("user" => "login", "pass" => "mot_de_passe", "key" => "cle_api"));
  • $countryCl->getCountries();
?>

La passation de commande se déroule avec le code ISO, représenté dans le tableau $countries par la clé "code".

Regardez aussi les exemples : récupération des pays.

3. Comment obtenir un devis ?

Voici les éléments dont on a besoin pour obtenir un devis :

  • le type d'envoi (encombrant, colis, palette, pli)
  • l'identifiant de la catégorie de contenu
  • le pays, la ville et le type de l'expéditeur
  • le pays, la ville et le type du destinataire
  • la date d'enlèvement (sauf dimanche et jours fériés)
<?php
  • $from = array("pays" => "FR", "code_postal" => "44000", "adresse" => "1, rue Racine", "type" => "particulier");
  • $to = array("pays" => "FR", "code_postal" => "33000", "adresse" => "1, Rue des Faures", "type" => "particulier");
  • $quotInfo = array("collecte" => "2011-05-11", "delai" => "aucun", "content_code" => 10120);
  • $cotCl = new Env_Quotation(array("user" => "login", "pass" => "mot_de_passe", "key" => "cle_api"));
  • $cotCl->setPerson("expediteur", $from);
  • $cotCl->setPerson("destinataire", $to);
  • $cotCl->setType("colis", array("poids" => 2, "longueur" => 30, "largeur" => 44, "hauteur" => 44));
  • $cotCl->getQuotation($quotInfo);
  • $cotCl->getOffers(false);
?>

Avant d'appeler getOffers() vous pouvez vous rassuer que la requête s'est correctement exécutée. Pour cela il faut que $curlError et $respError aient comme valeur false.

Regardez aussi les exemples : le devis Nantes - Bordeaux.

4. Comment passer une commande ?

L'API de l'EnvoiMoinsCher ne supporte que l'envoi d'un seul élément. Il est donc impossible d'expédier deux colis différents ou deux types d'envoi différents (par exemple un encombrant et un pli).

La passation d'une commande se déroule de la même manière que l'obtention d'une cotation. La seule différence consiste en informations que vous devez fournir en plus. Pour l'expéditeur et le destinataire vous devez indiquer leurs adresses exactes, numéros de téléphones, données personnelles. Concernant les données de l'envoi, en fonction de l'opérateur et du service choisis, vous pouvez être amenés à fournir des informations comme les disponibilités d'enlèvement, les points relais de dépôt et de retrait ou le montant d'assurance.

<?php
  • $from = array("pays" => "FR", "code_postal" => "75002", "type" => "particulier", "ville" => "Paris", "adresse" => "41, rue Saint Augustin| 3e étage", "civilite" => "M", "prenom" => "Prénom_expéditeur", "nom" => "Nom_expéditeur", "email" => "dev@boxtale.com", "tel" => "0601010101", "infos" => "Une information supplémentaire");
  • $to = array("pays" => "FR", "code_postal" => "13002", "type" => "particulier", "ville" => "Marseille", "adresse" => "1, rue Saint-Thome", "civilite" => "Mme", "prenom" => "Mme_expéditeur", "nom" => "Mme_expéditeur nom", "email" => "dev@boxtale.com", "tel" => "0601010101", "infos" => "");
  • $quotInfo = array("collecte" => "2011-05-11", "delai" => "aucun", "content_code" => 10120, "operateur" => "UPSE","disponibilite.HDE" => "09:00", "disponibilite.HLE" => "19:00", "prix_max_ttc" => 50);
  • $cotCl = new Env_Quotation(array("user" => "login", "pass" => "mot_de_passe", "key" => "cle_api"));
  • $cotCl->setPerson("expediteur", $from);
  • $cotCl->setPerson("destinataire", $to);
  • $cotCl->setType("colis", array("poids" => 2, "longueur" => 30, "largeur" => 44, "hauteur" => 44));
  • $orderPassed = $cotCl->makeOrder($quotInfo, true);
?>

Dans cet exemple, on passera une commande chez l'opérateur UPS à condition qu'elle sera inférieure à 50 euros TTC (paramètre "prix_max_ttc").

Regardez aussi les exemples : la commande Paris - Marseille, la commande Paris - Sydney, la commande Paris - Marseille avec l'assurance.

La fonction d'autoload.

Si vous utilisez un framework ou un système de gestion de contenu, vous pouvez rencontrer des problèmes avec l'inclusion automatique des classes de la librairie, comme c'est fait dans les fichiers d'exemple. Dans ce cas-là, vous pouvez être amenés à inclure les classes manuellement en utilisant require_once ou include_once avec les chemins respectifs vers les classes.

Pour plus d'informations sur les paramètres, la description des classes, veuillez-vous référer à la documentation de la librairie et de l'API.
